# AgentNexus 普通用户使用说明

> 面向**日常在项目里聊天、@ Bot、发文件**的用户，用大白话说明怎么用。不涉及建项目、加人、装系统等管理或部署操作。

---

## 一、先搞清楚几个词

| 词 | 什么意思 |
|----|----------|
| **工作空间** | 相当于“公司”或“组织”，是一个大容器。项目都挂在某个工作空间下面。 |
| **项目** | 相当于一个“协作群”或“频道”。大家在一个项目里聊天、@ Bot、发文件。系统里叫“频道”，这里都叫**项目**。 |
| **Bot** | 你 @ 的那个 AI 助手。只有被“加入”到当前项目的 Bot，你 @ 它才会回复。 |
| **成员** | 项目里可以有“人”和“Bot”；都要被加进项目后，才能在该项目里出现或回复。 |

---

## 二、怎么打开并进入项目

1. 用浏览器打开 AgentNexus 地址（一般是管理员给你的链接，例如 **http://你的服务器地址** 或 **http://localhost:80**）。
2. 当前版本没有登录页，会直接进主界面；左侧会显示「当前用户」和**项目列表**。
3. 在左侧点击某一个**项目名称**，右侧就会显示这个项目里的聊天记录，底部出现输入框。
4. 如果**左侧没有项目**：说明你还没有被加入任何项目，或系统里还没有建项目。请联系管理员把你加进项目，或让管理员先创建项目。

## 三、在项目里怎么用

### 3.0 不会用时问「引导 Bot」

在项目里输入 **@引导** 并加上你的问题，系统会根据说明书自动回复引导内容。例如：

- **@引导 怎么创建项目** → 会告诉你创建工作空间和项目的步骤  
- **@引导 怎么加入项目** → 会说明如何被管理员加进项目  
- **@引导 怎么接入 OpenClaw** → 会说明注册 Bot、加入项目的三步  
- **@引导 怎么用** 或 **@引导 帮助** → 会列出所有可问的问题  
- **@引导 Orchestrator 是什么** → 会说明 Orchestrator 的用法  

只有当前项目里已加入「引导」这个 Bot 时才能用（部署时若使用了种子数据，测试项目里已自带 @引导）。

### 3.0.1 Orchestrator：业务问答与建议

**Orchestrator** 是系统内置的业务问答 Bot（@coordinator）。它有两种工作方式：

1. **直接回答**（需管理员开启）：当你发消息**没有 @ 任何人**时，Orchestrator 会优先回答你的业务问题；若问题属于系统使用，会建议你 @引导。
2. **显式 @**：在消息里写 **@coordinator**，Orchestrator 会聚合频道内其他 Bot 的回复，或根据问题建议你 @ 某个部门 Bot。

若管理员开启了「自动接手」，Orchestrator 在回复中写「建议 @xxx 回答 XXX」时，被建议的 Bot 会自动接手回答，你会看到「@xxx 正在处理...」的提示。

### 3.1 发消息

- 在底部**输入框**里输入文字，点**「发送」**或按 **Enter**，消息就会发到这个项目里。
- 所有在这个项目里的成员（人和 Bot）都能看到。

### 3.2 让 Bot 回复你（@ Bot）

- 在消息里**写上 @Bot 的名字**（例如 `@引导`、`@coordinator`），系统会把这条消息转给这个 Bot。
- Bot 的回复会**以消息形式**出现在同一条聊天里，并带有「Bot」标识。
- **注意**：只有已经**被加入这个项目**的 Bot，你 @ 它才会生效；且 @ 的名字要和 Bot 的称呼完全一致（不能多空格或写错字）。

### 3.2.1 在聊天里把 Bot 拉进群（Slack 风格）

- 输入 **@** 时，会弹出 Bot 列表：**已在频道的排前面**，**未加入的排后面**（灰色显示「未加入」）。
- 若你选择了一个**未加入**的 Bot，系统会提示「是否邀请加入？」；点击**「加入并 @ta」**后，该 Bot 会加入本频道，输入框会自动插入 @Bot名，你可继续输入问题并发送，**不打断提问流程**。
- 此方式无需去「管理」→「添加成员」填 ID，适合快速邀请 Bot 进群。

### 3.3 上传文件

- 当前支持 **.txt、.md、.docx** 三种格式。
- 上传后系统会转成可读格式，Bot 在回复时可以参考这些内容。
- 若界面上有“上传”按钮，直接选文件即可；没有的话可联系管理员通过其它方式上传。

## 四、常见问题（用户最关心的）

**Q：我打开网页，左边没有项目？**  
A：说明你还没有被加入任何项目，或系统里还没有创建项目。请联系管理员：把你加进已有项目，或先创建一个项目再把你加进去。

**Q：我 @ 了 Bot 但没反应？**  
A：可能原因：（1）这个 Bot 还没有被加入当前项目——可在输入 @ 时从列表选该 Bot，系统会提示「是否邀请加入？」确认即可拉进群；或联系管理员；（2）你 @ 的名字和 Bot 的名字不一致（问管理员这个项目里 Bot 的确切称呼）；（3）Bot 或后方服务暂时故障（联系管理员排查）。

**Q：发消息没 @ 任何人，谁会回答？**  
A：若管理员开启了「Orchestrator 直接回答」，且 Orchestrator 已加入当前项目，则由 Orchestrator 回答业务问题；否则无人回复。可 @coordinator 或 @引导 获取帮助。

**Q：怎么让别人也能进这个项目？**  
A：需要**管理员**在系统里把对方加为项目成员。你联系管理员即可。

**Q：更多帮助在哪看？**  
A：在项目里 **@引导** 并输入问题（如「怎么创建项目」「怎么用」），引导 Bot 会按说明书自动回复。安装、建项目、加人、加 Bot 等由管理员操作，详见 [系统管理说明书](系统管理说明书.md)。技术故障排查见 [技术排查Q&A](技术排查Q&A.md)。
