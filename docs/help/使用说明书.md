# Cheers User Manual

> **Language**: English | [中文](使用说明书.zh-CN.md)

This is the public documentation index for Cheers. English is the default language; Chinese mirrors use the `.zh-CN.md` suffix.

## Quick Start

1. Follow the [Deployment Guide](deployment.md) to start the stack (Docker Compose, Helm/Kubernetes, or from source).
2. Open `http://localhost` and sign in — the initial admin account is created from `ADMIN_PASSWORD` in `.env` on first startup.
3. Connect an external agent — there is no built-in bot. Run an ACP-capable agent (Claude, Codex, OpenCode, …) behind the ACP connector (`packages/cheers-acp-connector-rs`), which bridges it to the gateway and hands it channel tools through `packages/cheers-mcp-server`. Follow the [Local Bot Setup Guide](本地Bot配置指南.md) (host daemon) or the [Agent Bridge Integration Guide](AgentBridge接入指南.md) (UI flow and protocol).
4. Add the Bot to a channel, then `@`-mention it — for example `@mybot how do I use this?`.

## Open Source Notes

- Do not use the example secrets in `.env.example` in production.
- Do not commit `.env`, databases, logs, uploads, private keys, or tokens.
- Replace example domains with your own deployment domain.
- Work branch PRs target `develop`; `main` only receives merges from `develop`.
