# Cheers User Guide

> **Language**: English | [中文](普通用户使用说明.zh-CN.md)

This guide is for people who use Cheers to chat in projects, mention Bots, and share files. It avoids deployment and administration details.

## Key Concepts

| Term | Meaning |
|---|---|
| Workspace | A top-level organization container. Channels/projects belong to a workspace. |
| Project / Channel | A collaboration room where people and Bots chat. The UI may call it a project; the API calls it a channel. |
| Member | A user or Bot that has been added to a channel. |
| Bot | An AI assistant that can be mentioned with `@username`. It only replies in channels where it is a member. |
| Coordinator | The built-in assistant. Mention `@Coordinator` for help, clarification, routing suggestions, and business questions. |

## Entering a Project

1. Open the Cheers URL provided by your administrator, for example `http://localhost` in local development.
2. Sign in if the login page appears. Some local demo deployments may enter the workspace directly.
3. Select a workspace and channel from the left sidebar.
4. If no channel is visible, ask an administrator to create one and add you as a member.

## Getting Help from Coordinator

In a channel, send messages such as:

```text
@Coordinator how do I create a project?
@Coordinator how do I invite a Bot?
@Coordinator how do I connect a local ACP agent?
@Coordinator help
```

Coordinator can ask clarification questions when your request is incomplete. Fill in the clarification card and continue, or skip it if you want the Bot to answer the original question directly.

## Sending Messages

- Type a message in the composer and press Enter or click Send.
- Everyone in the channel can see channel messages.
- Mention a Bot with `@BotName` to route the message to that Bot.

## Inviting a Bot from Chat

When you type `@`, the mention picker shows Bots already in the channel first and Bots not yet joined after them. Selecting a Bot that is not in the channel asks whether to invite it. Confirming adds the Bot and inserts the mention without interrupting your question.

## Uploading Files

Use the upload button to attach supported files. Uploaded files can be referenced by Bots when they answer. Supported types depend on deployment configuration, but common text, Office, PDF, image, and archive types are supported by the default file pipeline and preview stack.

## Deleting And Cleanup

- **Messages, topics, and announcements**: hover a message and use delete. Users can delete their own messages; channel administrators and system administrators can delete non-system messages in the channel. Deleted messages remain as tombstones so replies and topic structure stay intact.
- **Files**: delete actions are available in the personal file library, channel Files panel, and file preview panel. If a file is still referenced by channel messages, delete those messages first; unreferenced files are removed from records and storage.
- **Avatars**: Clear removes managed uploaded avatar objects. External avatar URLs are only cleared from the profile.
- **Account**: account deletion is implemented as deactivation. It clears personal profile data, friendships, keychain items, and managed avatar data, then blocks future login while preserving historical message ownership references.

## Common Questions

**I cannot see any project.**
You are probably not a member of any channel yet, or no channel has been created. Ask an administrator to create a channel and add you.

**I mentioned a Bot but it did not reply.**
Check whether the Bot is in the current channel, whether the username is exact, and whether the Bot/provider is online. Contact an administrator if it still fails.

**Who answers if I do not mention anyone?**
If `auto_assist` is enabled for the channel and Coordinator is a member, Coordinator may answer. Otherwise no Bot is triggered.

**Where can I find more help?**
Ask `@Coordinator help`, or read the [Admin Guide](系统管理说明书.md) and [Troubleshooting Q&A](技术排查Q&A.md).
