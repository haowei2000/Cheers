# Cheers 普通用户使用说明

> **语言**：中文 | [English](普通用户使用说明.md)

> 面向**日常在项目里聊天、@ Bot、发文件**的用户，用大白话说明怎么用。不涉及建项目、加人、装系统等管理或部署操作。

---

## 一、先搞清楚几个词

| 词 | 什么意思 |
|----|----------|
| **工作空间** | 相当于“公司”或“组织”，是一个大容器。项目都挂在某个工作空间下面。 |
| **项目** | 相当于一个“协作群”或“频道”。大家在一个项目里聊天、@ Bot、发文件。系统里叫“频道”，这里都叫**项目**。 |
| **Bot** | 你 @ 的那个 AI 助手。只有被“加入”到当前项目的 Bot，你 @ 它才会回复。 |
| **成员** | 项目里可以有“人”和“Bot”；都要被加进项目后，才能在该项目里出现或回复。 |

---

## 二、怎么打开并进入项目

1. 用浏览器打开 Cheers 地址（一般是管理员给你的链接，例如 **http://你的服务器地址** 或 **http://localhost**）。
2. 如果出现登录页，使用管理员分配的账号登录；如果是本地体验环境，可能会直接进入主界面。
3. 左侧会显示你可访问的工作空间和**项目列表**。点击某一个**项目名称**，右侧就会显示这个项目里的聊天记录，底部出现输入框。
4. 如果**左侧没有项目**：说明你还没有被加入任何项目，或系统里还没有建项目。请联系管理员把你加进项目，或让管理员先创建项目。

## 三、在项目里怎么用

### 3.0 不会用时问「@Coordinator」

在项目里输入 **@Coordinator** 并加上你的问题，系统会根据说明书自动回复引导内容。例如：

- **@Coordinator 怎么创建项目** → 会告诉你创建工作空间和项目的步骤
- **@Coordinator 怎么加入项目** → 会说明如何被管理员加进项目
- **@Coordinator 怎么接入本地 ACP Agent** → 会说明安装 ACP agent npm 包、注册 Bot、加入项目的步骤；OpenClaw 入口已废弃/遗留
- **@Coordinator 怎么用** 或 **@Coordinator 帮助** → 会列出所有可问的问题
- **@Coordinator 能做什么** → 会说明内置协作助手的用法

只有当前项目里已加入内置 `Coordinator` 时才能用（部署时若使用了种子数据，默认项目里已自带）。

#### 3.0.1 Coordinator 的澄清机制

当你的问题比较模糊、或缺少关键信息时，Coordinator 不会直接猜测，而是先弹出一张**澄清卡片**，请你先补充信息，再继续回答：

- **触发时机**：问题太笼统（如「帮我搞一下」「怎么弄」）、关键参数没说清楚（比如“要创建项目，但没说在哪个工作空间”），或管理员配置为“更严格澄清模式”时。
- **澄清卡片内容**：
  - 以「为避免误解，我先确认几个问题」开头；
  - 下面是一组单选/多选题，有时会带「其他」输入框，请你补充说明（例如**其他工作空间名**、**具体类型**等）。
- **操作方式**：
  - 勾选/填写完后点 **「继续」**：系统会把你的选择整理成一条「澄清回答」发给 Coordinator，随后它会基于这些补充信息继续给出正式答复；
  - 若你觉得澄清没有必要，可以点 **「跳过」**，Coordinator 会按原问题直接回答。
- **展示效果**：澄清过程中产生的中间消息会集中显示，避免把主聊天刷得很乱。

### 3.0.2 Coordinator：业务问答与建议

**Coordinator** 是系统内置的统一协作 Bot。它有两种工作方式：

1. **直接回答**（需管理员开启或频道开启 auto_assist）：当你发消息**没有 @ 任何人**时，Coordinator 会优先回答你的业务问题。
2. **显式 @**：在消息里写 **@Coordinator**，它会聚合频道内其他 Bot 的回复，或根据问题建议你 @ 某个部门 Bot。

若管理员开启了「自动接手」，Coordinator 在回复中写「建议 @xxx 回答 XXX」时，被建议的 Bot 会自动接手回答，你会看到「@xxx 正在处理...」的提示。

### 3.1 发消息

- 在底部**输入框**里输入文字，点**「发送」**或按 **Enter**，消息就会发到这个项目里。
- 所有在这个项目里的成员（人和 Bot）都能看到。

### 3.2 让 Bot 回复你（@ Bot）

- 在消息里**写上 @Bot 的名字**（例如 `@Coordinator`），系统会把这条消息转给这个 Bot。
- Bot 的回复会**以消息形式**出现在同一条聊天里，并带有「Bot」标识。
- **注意**：只有已经**被加入这个项目**的 Bot，你 @ 它才会生效；且 @ 的名字要和 Bot 的称呼完全一致（不能多空格或写错字）。

### 3.2.1 在聊天里把 Bot 拉进群（Slack 风格）

- 输入 **@** 时，会弹出 Bot 列表：**已在频道的排前面**，**未加入的排后面**（灰色显示「未加入」）。
- 若你选择了一个**未加入**的 Bot，系统会提示「是否邀请加入？」；点击**「加入并 @ta」**后，该 Bot 会加入本频道，输入框会自动插入 @Bot名，你可继续输入问题并发送，**不打断提问流程**。
- 此方式无需去「管理」→「添加成员」填 ID，适合快速邀请 Bot 进群。

### 3.3 上传文件

- 当前支持 **.txt、.md、.docx** 三种格式。
- 上传后系统会转成可读格式，Bot 在回复时可以参考这些内容。
- 若界面上有“上传”按钮，直接选文件即可；没有的话可联系管理员通过其它方式上传。

### 3.4 删除与清理

- **消息/话题/公告**：悬停消息后点击删除。普通用户可删除自己发送的消息；频道管理员和系统管理员可删除频道内非系统消息。删除后聊天中保留“已删除”占位，避免回复和话题结构断裂。
- **文件**：个人文件库、频道 Files 面板和文件预览面板都有删除入口。若文件仍被频道消息引用，系统会提示先删除相关消息；没有活跃引用时会清理文件记录和存储对象。
- **头像**：个人设置里的 Clear 会删除系统托管头像对象；如果头像是外部 URL，则只清空链接。
- **账号**：个人设置提供删除账号入口。账号删除采用停用方式，会清理个人资料、好友关系、Keychain 和头像，并禁止再次登录；历史消息保留为已停用用户的归属记录。

## 四、常见问题（用户最关心的）

**Q：我打开网页，左边没有项目？**
A：说明你还没有被加入任何项目，或系统里还没有创建项目。请联系管理员：把你加进已有项目，或先创建一个项目再把你加进去。

**Q：我 @ 了 Bot 但没反应？**
A：可能原因：（1）这个 Bot 还没有被加入当前项目——可在输入 @ 时从列表选该 Bot，系统会提示「是否邀请加入？」确认即可拉进群；或联系管理员；（2）你 @ 的名字和 Bot 的名字不一致（问管理员这个项目里 Bot 的确切称呼）；（3）Bot 或后方服务暂时故障（联系管理员排查）。

**Q：发消息没 @ 任何人，谁会回答？**
A：若管理员开启了直接回答（或频道开启 auto_assist），且 `Coordinator` 已加入当前项目，则由它回答业务问题；否则无人回复。你也可以直接 `@Coordinator` 获取帮助。

**Q：怎么让别人也能进这个项目？**
A：需要**管理员**在系统里把对方加为项目成员。你联系管理员即可。

**Q：更多帮助在哪看？**
A：在项目里 **@Coordinator** 并输入问题（如「怎么创建项目」「怎么用」），内置 Bot 会按说明书自动回复。安装、建项目、加人、加 Bot 等由管理员操作，详见 [系统管理说明书](系统管理说明书.md)。技术故障排查见 [技术排查Q&A](技术排查Q&A.md)。
