# 安装部署说明（已迁移）

> **语言**：中文 | [English](安装部署说明.md)

> ⚠️ **本文档已迁移。** 旧内容描述的是已移除的 Python 后端（Alembic 迁移、
> `uvicorn`、`SEED_DATA`、内置 Coordinator bot），与当前代码不符。后端现在是
> 单一的 Rust gateway，启动时自动执行 `sqlx` 迁移。

请使用以下最新且准确的指南：

- **Docker Compose（单机）** —— [docker-compose-deploy.zh-CN.md](docker-compose-deploy.zh-CN.md) / [English](docker-compose-deploy.md)
  环境要求、必需的 RS256 JWT 密钥对、`.env`、启动核心栈、可选 OpenCode bot、TLS 与运维。
- **Kubernetes（Helm）** —— [../../deploy/helm/cheers/README.md](../../deploy/helm/cheers/README.md)
  chart values、secrets、ingress，以及本地 kind 流程。

与旧文档不同的要点：

- **迁移** 在 gateway 启动时自动执行（`sqlx`）；没有 Alembic、没有 `db-migrate`
  服务，也没有手动迁移步骤。
- **认证** 使用 RS256 密钥对：设置 `JWT_PRIVATE_KEY` 和 `JWT_PUBLIC_KEY`
  （不是 `JWT_SECRET_KEY`）。用 `.env.example` 中的 `openssl` 命令生成。
- **初始化** 仅创建管理员，且仅当 `ADMIN_PASSWORD` 设为非示例值时。没有默认
  workspace/channel/prompt，也没有 Coordinator bot。
- **健康检查** `GET /health` 返回纯文本 `ok`。
