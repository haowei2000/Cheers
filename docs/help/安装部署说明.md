# Installation Guide (moved)

> **Language**: English | [中文](安装部署说明.zh-CN.md)

> ⚠️ **This guide has moved.** The previous content described the old Python
> backend (Alembic migrations, `uvicorn`, `SEED_DATA`, a built-in Coordinator
> bot) and no longer matches the codebase. The backend is now a single Rust
> gateway that runs its `sqlx` migrations automatically on startup.

Use the current, accurate guides instead:

- **Docker Compose (single host)** — [docker-compose-deploy.md](docker-compose-deploy.md) / [中文](docker-compose-deploy.zh-CN.md)
  Prerequisites, the required RS256 JWT keypair, `.env`, bringing up the core
  stack, the optional OpenCode bot, TLS, and operations.
- **Kubernetes (Helm)** — [../../deploy/helm/cheers/README.md](../../deploy/helm/cheers/README.md)
  Chart values, secrets, ingress, and the local kind workflow.

Quick facts that differ from the old guide:

- **Migrations** run automatically when the gateway starts (`sqlx`); there is no
  Alembic, no `db-migrate` service, and no manual migration step.
- **Auth** uses an RS256 keypair: set `JWT_PRIVATE_KEY` and `JWT_PUBLIC_KEY`
  (not `JWT_SECRET_KEY`). Generate with the `openssl` commands in `.env.example`.
- **Seeding** creates only the administrator, and only when `ADMIN_PASSWORD` is
  set to a non-example value. There is no default workspace/channel/prompt or
  Coordinator bot.
- **Health check** `GET /health` returns the plain text `ok`.
