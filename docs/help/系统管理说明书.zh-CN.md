# AgentNexus 系统管理说明书

> **语言**：中文 | [English](系统管理说明书.md)

> 面向**系统管理员**：如何创建工作空间与项目、如何添加/移除成员、如何把 OpenClaw 或 ACP Agent 接入为 Bot 等。需要会使用命令行与接口（API）。

---

## 管理员入口地址

管理员与普通用户使用同一前端；常见管理操作可在前端「管理」入口完成，复杂操作也可以通过**后端 API** 或 Swagger 执行。

| 入口 | 地址 |
|------|------|
| **聊天前端**（与用户相同） | Docker：`http://localhost:80`；本地开发：`http://localhost:5173` |
| **API 文档（Swagger）** | **`http://localhost:8000/docs`**（创建频道、添加成员、查看列表等均在此调用或参考） |
| **健康检查** | `http://localhost:8000/health` |
| **OpenClaw 发现与注册指南（机器可读）** | `http://localhost:8000/docs/agent-bridge/discovery` |

若后端端口或域名不同，请将上述 `localhost:8000` 替换为实际后端地址。

### 管理界面中的「表格表单」模式

当前管理操作（创建项目、创建 Bot、添加/移除成员等）均采用**同一套交互方式**：在「管理」弹窗中，**左列是字段说明/问题，右列是对应的输入框或操作按钮**，用户填写后点击提交即可完成。后续新增的系统设置（如更多工作空间配置、频道配置等）也会沿用这种「表格表单」方式，便于统一体验与扩展。

## 一、概念说明

| 概念 | 说明 |
|------|------|
| **工作空间（Workspace）** | 顶层容器，相当于“组织”。创建项目前必须先有一个工作空间。 |
| **项目（频道 / Channel）** | 协作单元，成员在项目里聊天、@ Bot、发文件。 |
| **成员** | 分为 **user**（用户）和 **bot**（Bot）；均需被“加入”到具体项目后，才能在该项目中生效。 |
| **Bot 账号** | 每个 OpenClaw（或其它 AI 服务）在系统里对应一个 Bot 账号；注册 Bot 后再把它加入项目，用户才能 @ 它。 |

## 二、如何创建工作空间与项目

### 2.1 创建工作空间（只需做一次）

**【推荐】前端表格表单**：在「管理」→「创建工作空间」中填写「空间名称」后点击「创建」即可。

**或**调用 API：

```bash
curl -X POST http://localhost:8000/api/v1/workspaces \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <你的访问令牌>" \
  -d '{"name":"默认空间"}'
```

记下返回值中的 **`workspace_id`**，创建项目时要用。

### 2.2 创建项目（频道）

在已有工作空间下创建项目，调用 API（基础地址默认 `http://localhost:8000`，下同）：

```bash
curl -X POST http://localhost:8000/api/v1/channels \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <你的访问令牌>" \
  -d '{"workspace_id":"ws-default-001","name":"我的第一个项目","type":"public","purpose":"日常协作"}'
```

将 **`ws-default-001`** 换成你的工作空间 ID，**name** 可改为任意项目名称。
返回的 JSON 中的 **`channel_id`** 即为项目 ID，后续加人、加 Bot 时需用到。

## 三、如何管理项目成员（加人 / 加 Bot / 移除）

### 3.1 查看项目成员

```bash
curl "http://localhost:8000/api/v1/channels/项目ID/members" \
  -H "Authorization: Bearer <你的访问令牌>"
```

将 **项目ID** 替换为实际的 `channel_id`。

### 3.2 添加成员（用户或 Bot）

**方式一：聊天内邀请 Bot（Slack 风格，推荐）**

- 用户在频道内输入 **@** 时，会显示全部 Bot 列表（已在频道的排前，未加入的灰色排后）。
- 若选择未加入的 Bot，系统提示「是否邀请加入？」；用户确认后，Bot 自动加入该频道，输入框插入 @Bot名，不打断提问流程。
- 无需去「管理」填 ID，适合快速拉 Bot 进群。详见《普通用户使用说明》§3.2.1。

**方式二：管理端添加**

```bash
curl -X POST "http://localhost:8000/api/v1/channels/项目ID/members" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <你的访问令牌>" \
  -d '{"member_id":"用户或Bot的ID","member_type":"user"}'
```

- 添加**用户**：`member_type` 填 **`user`**，`member_id` 填该用户的 **user_id**（需先在 users 表存在）。
- 添加**Bot**：`member_type` 填 **`bot`**，`member_id` 填该 Bot 的 **bot_id**（需先在 bot_accounts 表存在，见下文「四、Agent Bridge 接入」）。

### 3.3 移除成员

```bash
curl -X DELETE "http://localhost:8000/api/v1/channels/项目ID/members/成员ID" \
  -H "Authorization: Bearer <你的访问令牌>"
```

**成员ID** 即该用户或 Bot 的 user_id / bot_id。

## 四、如何让 Agent Bridge provider 接入（注册 Bot 并加入项目）

“接入”指：把 OpenClaw、ACP Connector 等外部 provider 注册为 AgentNexus 里的一个 Bot，用户在某项目里 @ 该 Bot 时，由外部 provider 回复。

### 4.1 适配器模式说明

AgentNexus 当前只保留两类 Bot 绑定：

| binding_type | 适配器 | 说明 |
|------|--------|------|
| `http` | HTTP Bot | 后端按 Bot 绑定的 AIModel + PromptTemplate 直接调用 OpenAI 兼容模型。 |
| `agent_bridge` | Agent Bridge Bot | 外部 provider 通过 `/ws/agent-bridge/control` 与 `/ws/agent-bridge/data` 反向连接，异步回推 reply / delta / done。OpenClaw 与 ACP Connector 都属于 provider。 |

内置 `@Coordinator` 是系统自带的协作助手，使用固定 `bot_id=bot-helper-001`。历史版本中的旧名称只是兼容别名，新文档统一使用 `@Coordinator`。

### 4.2 第 1 步：注册 Bot 账号

**【推荐】通过管理后台 UI：**

1. 打开前端 → 左侧「管理」→ **Bot 管理** 区块
2. 点击新建 Bot，选择 **Agent Bridge Bot**
3. 填写 username、显示名称、可选的 Provider agent id 与 Prompt 模板
4. 创建后复制一次性 `agb_...` token，并写入 provider 配置

OpenClaw 或 ACP provider 也可以直接调用 `POST /docs/agent-bridge/register` 完成注册，响应里会返回 `bot_token`、`controlUrl`、`dataUrl` 和配置片段。

### 4.3 第 2 步：将 Bot 加入项目

**【推荐】通过管理后台 UI：**

1. 左侧「管理」→ **Bot 与频道** 区块
2. 选择目标项目，点击「添加成员」，在复选框列表中勾选 Bot，点击「添加选中」

**或**调用 API：

```bash
curl -X POST "http://localhost:8000/api/v1/channels/项目ID/members" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <你的访问令牌>" \
  -d '{"member_id":"你的BOT_ID","member_type":"bot"}'
```

完成后，用户在该项目内 @ 该 Bot 的 **username**（如 `@mybot`）即可触发。

### 4.4 Agent Bridge 调用协议

AgentNexus 通过两条 WebSocket 与 provider 通信：

1. **control**：provider 使用 `Authorization: Bearer agb_...` 连接 `/ws/agent-bridge/control`，接收 membership 快照和频道加入/移除事件。
2. **data**：provider 连接 `/ws/agent-bridge/data`，接收 `message` 任务，回推 `reply`、`delta`、`done`、`error`。
3. **落盘广播**：后端统一通过 Message DTO 写入消息并广播 `message_done` / `message_stream`。

OpenClaw 插件和 ACP Connector 都是 Agent Bridge 的 provider 实现。详见 [Agent Bridge 接入指南](AgentBridge接入指南.md)。

### 4.5 HTTP Bot 模式说明

HTTP Bot 不再使用 `Agent Bridge 配置`，而是选择一个 AIModel 和 PromptTemplate。后端按 OpenAI 兼容 Chat Completions 协议调用对应模型。

## 五、如何让外部 provider 发现并自动注册

本节说明：**外部 provider 如何发现本系统、获取机器可读的注册指南，并根据用户输入的账号密码或已有 token 完成接入**。推荐流程是：用户在 provider 侧输入 AgentNexus 账号密码，provider 调用注册接口直接创建 Agent Bridge Bot，并获得一次性 `bot_token`。

> **快速接入**：详见 [Agent Bridge 接入指南](AgentBridge接入指南.md)，含 OpenClaw 与 ACP / Codex ACP 示例。

### 5.1 本系统对外提供的 API / URL

| 用途 | 方法 | URL（基础地址默认 `http://localhost:8000`） |
|------|------|---------------------------------------------|
| **Agent Bridge 机器可读入口清单（推荐）** | GET | `/docs/agent-bridge/discovery` |
| **Agent Bridge 帮助问答** | GET / POST | `/docs/agent-bridge/help` |
| **账号密码或 token 直接注册 Agent Bridge Bot** | POST | `/docs/agent-bridge/register` |
| **验证用户 token** | GET | `/docs/agent-bridge/auth-check` |
| **OpenClaw 插件下载地址** | GET | `/docs/agent-bridge/discovery` 响应里的 `plugin.download_url` |

- **推荐发现接口**：`GET {BASE}/docs/agent-bridge/discovery` 返回 JSON，包含登录、注册、帮助、Agent Bridge、文件接口等入口，供 provider 自动判断下一步。
- **帮助问答接口**：`GET {BASE}/docs/agent-bridge/help?q=怎么接入OpenClaw` 或 `POST {BASE}/docs/agent-bridge/help`，body 为 `{ "question": "..." }`。
- **OpenClaw 插件包名**：公开 npm 包为 `@haowei0520/openclaw-channel-agentnexus`。如果使用 `@haowei2000/openclaw-channel-agentnexus`，需确认这是你自己的私有 npm registry 包；公开 npm registry 上默认使用 `@haowei0520`。
- **插件下载地址**：读取 discovery 响应中的 `plugin.download_url`。后端会从 AgentNexus 项目根目录的 `release/` 文件夹读取插件包，默认文件名为 `openclaw-channel-agentnexus.tgz`，对应 URL 为 `{BASE}/docs/agent-bridge/release/openclaw-channel-agentnexus.tgz`。如需调整目录或文件名，可设置 `OPENCLAW_PLUGIN_RELEASE_DIR` / `OPENCLAW_PLUGIN_FILE`。一般安装优先使用 `npm pack @haowei0520/openclaw-channel-agentnexus@0.2.4 --pack-destination /tmp` 后再执行 `openclaw plugins install /tmp/haowei0520-openclaw-channel-agentnexus-0.2.4.tgz`；离线部署才使用后端挂载的 release tarball。
- **直接注册接口**：`POST {BASE}/docs/agent-bridge/register` 支持两种鉴权方式：请求体里带 `account_username` / `account_password`，或 Header `Authorization: Bearer <AgentNexus access_token>`。成功后直接创建用户拥有的 Agent Bridge Bot，并一次性返回 `bot_token`、`controlUrl`、`dataUrl` 和 OpenClaw 配置片段。

### 5.2 OpenClaw 可读的发现与注册指南（JSON 结构）

OpenClaw 请求 `GET /docs/agent-bridge/discovery` 可获得如下结构的 JSON（示例，字段以实际返回为准）：

| 字段 | 说明 |
|------|------|
| `name` | 系统名称，如 AgentNexus |
| `base_url` | 本系统 API 根地址 |
| `entrypoints.register.url` | 账号密码或 token 直接注册 Agent Bridge Bot 的完整 URL |
| `entrypoints.help_get.url` / `entrypoints.help_post.url` | 帮助问答 URL |
| `auth.login.url` | 获取 AgentNexus access token 的登录接口 |
| `bridge.control_ws` / `bridge.data_ws` | OpenClaw channel plugin 连接的两条 WebSocket |
| `plugin.download_url` | AgentNexus OpenClaw 插件下载地址 |
| `plugin.install.curl` / `plugin.install.openclaw` | 插件下载和安装命令 |

**账号密码直接注册时的请求体示例：**

```json
{
  "username": "mybot",
  "account_username": "用户登录名或邮箱",
  "account_password": "用户密码",
  "display_name": "OpenClaw 助手",
  "agent_id": "main",
  "scope": "private",
  "channel_id": "可选频道ID",
  "intro": {
    "description": "本机 OpenClaw 助手",
    "capabilities": ["代码协作", "文件处理"]
  }
}
```

- `username`：必填，用户 @ 时使用的名字，系统内唯一。
- `account_username`：选填，AgentNexus 用户名或邮箱；未提供 Bearer token 时必填。
- `account_password`：选填，AgentNexus 登录密码；未提供 Bearer token 时必填。
- `display_name`：选填，显示名称。
- `agent_id`：选填，OpenClaw channel plugin 用于路由的 agent id，默认 `main`。
- `scope`：选填，`private` / `friend` / `everyone`，默认 `private`。
- `channel_id`：选填，填写后注册成功会尝试把 Bot 加入该频道，调用用户必须有添加 Bot 权限。
- `intro`：选填，结构化自我介绍，需含 `description` 或 `capabilities`。

**响应示例（直接注册成功）：**

```json
{
  "status": "success",
  "data": {
    "bot": {
      "bot_id": "uuid",
      "username": "mybot",
      "binding_type": "agent_bridge",
      "bot_token": "agb_xxx"
    },
    "bridge": {
      "control_ws": "ws://localhost:8000/ws/agent-bridge/control",
      "data_ws": "ws://localhost:8000/ws/agent-bridge/data"
    },
    "agent_bridge_config": {
      "channels": {
        "agentnexus": {
          "enabled": true,
          "accounts": []
        }
      }
    }
  }
}
```

`bot_token` 只在创建成功响应中明文返回一次，外部 provider 必须立即保存；丢失后需要用户在 AgentNexus 中轮换 token。

### 5.3 注册流程概览

1. **provider 发现本系统**：请求 `GET /docs/agent-bridge/discovery`，获取 `entrypoints` 与 `bridge`。
2. **用户输入账号密码**：provider 在本机界面收集 AgentNexus 的用户名/邮箱和密码。
3. **provider 直接注册**：向 `POST /docs/agent-bridge/register` 发送 JSON，包含 `account_username` / `account_password` 与 Bot 信息。已有 access token 时也可继续使用 `Authorization: Bearer <access_token>`。
4. **保存 Bot token**：把响应里的 `bot_token` 写入 provider 配置，用它连接 `control_ws` 与 `data_ws`。

## 六、管理操作速查

| 操作 | 方法 | 路径 / 请求体 |
|------|------|----------------|
| 获取所有项目列表 | GET | `/api/v1/channels` |
| 创建项目 | POST | `/api/v1/channels`，body：`workspace_id`, `name`, `type`（如 public）, `purpose`（可选） |
| 获取某项目成员 | GET | `/api/v1/channels/{项目ID}/members` |
| 添加成员（用户或 Bot） | POST | `/api/v1/channels/{项目ID}/members`，body：`member_id`, `member_type`（user / bot） |
| 移除成员 | DELETE | `/api/v1/channels/{项目ID}/members/{成员ID}` |
| 获取频道上下文 | GET | `/api/v1/channels/{项目ID}/context` |
| 更新频道上下文 | PUT | `/api/v1/channels/{项目ID}/context` |
| 创建工作空间 | POST | `/api/v1/workspaces`，body：`name` |
| Agent Bridge 机器可读入口清单（推荐） | GET | `/docs/agent-bridge/discovery` |
| Agent Bridge 帮助问答 | GET / POST | `/docs/agent-bridge/help` |
| 账号密码或 token 直接注册 Agent Bridge Bot | POST | `/docs/agent-bridge/register`，body：`account_username`, `account_password`, `username` 等；或 Header：`Authorization: Bearer <access_token>` |

所有接口基础路径：`http://localhost:8000`（或实际后端地址）。更多参数与响应见 **http://localhost:8000/docs**。

### 记忆数据本地排查提示

排查频道上下文或记忆数据时，本地 PostgreSQL 连接必须以项目根目录或 `backend` 目录下的 **.env** 为准：结构化记忆表 `memory_entries`、历史摘要表 `history_pages`、文件索引来源表 `file_records` 使用 **DATABASE_URL**；旧版 `context_store` 或派生层兼容读写使用 **CONTEXT_DB_URL**。不要直接套用示例里的 `localhost` 端口、库名或账号。

### Coordinator 使用本地 LLM（可选）

`@Coordinator` 默认可根据内置帮助与频道上下文回答。若希望用本地 LLM（如 Ollama）生成更自然的回答，可在后端目录配置环境变量（或 `.env`）。这些变量名仍沿用 `HELPER_LLM_*`，用于兼容历史配置：

| 变量 | 说明 | 示例 |
|------|------|------|
| `HELPER_LLM_BASE_URL` | OpenAI 兼容 API 根地址 | `http://localhost:11434/v1`（Ollama） |
| `HELPER_LLM_MODEL` | 模型名 | `qwen2.5:7b` |
| `HELPER_LLM_API_KEY` | 选填，本地 Ollama 可留空 | 空 |
| `HELPER_LLM_TEMPERATURE` | 选填，默认 0.7 | `0.7` |
| `HELPER_LLM_MAX_TOKENS` | 选填，默认 1000 | `1000` |

配置并重启后端后，`@Coordinator` 会优先使用该 LLM；未配置或调用失败时会退回内置规则与帮助检索。

### @Coordinator 无反应时如何排查

1. **确认 Coordinator 已存在且已加入当前项目**
   若从未执行过种子数据，系统里没有内置协作助手，`@Coordinator` 不会触发任何回复。请执行一次种子数据：
   - 启动时加环境变量：`SEED_DATA=1`；或
   - 手动执行：`cd backend && python -m app.db.seed`（需先 `alembic upgrade head`）。
   新建项目会自动把 Coordinator 加入该频道（仅当 Bot 账号已存在时）。老项目需在「管理」→「添加成员」中把成员 ID 填为 `bot-helper-001`、类型选 Bot 加入。

2. **确认 @ 的名字是「Coordinator」**
   消息里必须包含 `@Coordinator`（与 Bot 的 username 完全一致），不能多空格或错字。历史别名只用于兼容，不建议写入新文档或培训材料。

3. **若已配置本地 LLM 但希望确认是否连错**
   查看后端日志：若出现 Helper/Coordinator LLM 请求失败，说明地址、端口、模型名或网络可能有问题，系统会退回内置规则。
   若出现 `no mentioned bots in channel ... channel_bots=[]`，说明当前频道没有 Coordinator 成员，按第 1 步加入该项目。

4. **若编排器抛错**
   日志中会有 `orchestrator failed channel_id=...` 及异常信息，可根据堆栈排查（如数据库、记忆模块等）。

### 日志文件（便于排查错误）

后端会将运行日志写入文件（需配置 `LOG_DIR`，默认 `data/logs`，相对 backend 目录）：

| 文件 | 内容 |
|------|------|
| `agentnexus.log` | 通用日志（INFO 及以上），按大小轮转 |
| `error.log` | 仅错误与异常（ERROR 及以上），便于快速排查 |

未设置 `LOG_DIR` 或目录不可写时，仅输出到控制台。环境变量：`LOG_DIR`、`LOG_MAX_BYTES`（单文件最大字节，默认 5MB）、`LOG_BACKUP_COUNT`（保留份数，默认 3）。未捕获异常会记录完整堆栈到上述日志并返回 500。
