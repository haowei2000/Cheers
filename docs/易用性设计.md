# AgentNexus 易用性设计

> 面向「低计算机能力用户」的易用第一原则（需求汇总 §3、§4），在界面、流程与文档上统一约束与扩展点。
>
> **版本**：v1.1（2026-03-11）
>
> **更新说明**：新增第 10 章「多人协作易用性设计」与第 11 章「MVP 功能差异分析」

## 1. 设计目标与原则

| 目标 | 说明 |
|------|------|
| 零文档上手 | 核心操作（进频道、发消息、@Bot）无需阅读手册即可完成 |
| 向导优先 | 管理类操作（创建空间/频道、添加 Bot）优先提供向导式步骤，M2 管理后台落地 |
| 反馈即时 | 操作结果、错误原因以简短中文提示，避免技术术语堆砌 |
| 一致用语 | 全系统统一使用「频道」「Bot」「空间」等术语，与《使用说明书》一致 |

## 2. 用户角色与使用场景

| 角色 | 主要场景 | 易用性重点 |
|------|----------|------------|
| 普通成员 | 在频道内看消息、发消息、@Bot 提问、上传文件 | 界面类 Slack，输入框占位符提示「输入消息，@Bot 可触发回复」；Bot 回复与用户消息视觉区分 |
| 频道/空间管理员 | 创建频道、拉人/拉 Bot 进频道 | M1 可通过 API 完成；M2 提供 Web 向导（选择空间 → 填频道名 → 选择成员/Bot） |
| 系统管理员 | 部署、配置环境变量、首次初始化 Workspace、查看日志 | 提供 .env.example 与《使用说明书》安装/配置/故障排除章节；M2 提供健康检查与初始化引导页 |

## 3. 首次使用与引导

### 3.1 首次部署后

- **数据初始化**：当前需先执行数据库迁移（`alembic upgrade head`），并至少创建一个 Workspace（见《使用说明书》§ 首次使用）。M2 可提供「首次访问引导页」：检测无 Workspace 时引导创建默认空间与示例频道。
- **空状态**：频道列表为空时，界面提示「暂无频道，请联系管理员创建」或提供「创建频道」入口（M2 向导）。
- **空频道**：进入频道后无消息时，提示「在此输入消息，或 @Bot 名称 让 AI 回复」，避免空白困惑。

### 3.2 新成员进入频道

- 左侧频道列表展示当前用户有权限的频道，选中即加载历史消息并建立 WebSocket 实时收信。
- 无需「加入频道」二次确认（当前模型为成员由管理员添加），降低步骤数。

## 4. 界面与交互

### 4.1 布局与视觉

- **左侧边栏**：固定展示「当前用户」与「频道」列表；当前选中频道高亮（如蓝色背景）。
- **主区**：上方为消息流（用户消息与 Bot 消息区分样式），下方为输入框 + 发送按钮。
- **Bot 消息**：与用户消息在背景色或标签上区分（如「Bot」标签、浅绿背景），便于扫读。

### 4.2 输入与发送

- 输入框占位符：`输入消息，@Bot 可触发回复...`，提示核心能力。
- 发送：支持 Enter 发送（Shift+Enter 换行若后续实现）；发送按钮文案统一为「发送」。
- @ 提及：M2 可做 @ 自动补全（输入 `@` 后列出频道内 Bot 与成员），减少记忆负担。

### 4.3 文件上传

- M1 支持 .txt / .md / .docx；格式限制在接口层返回明确错误（如「仅支持 .txt、.md、.docx」）。
- 上传后可查询状态（转换中 / 已就绪 / 失败）；前端可展示「文件处理中，请稍候」或「已就绪，Bot 可引用」等简短状态（M2 集成到消息流更佳）。

## 5. 错误与反馈

### 5.1 接口错误

- 统一 JSON 结构：`{"status": "error", "message": "简短原因", "data": null}`；避免向用户展示堆栈或内部 key。
- 常见 HTTP 状态与前端建议：
  - 404（频道/文件不存在）：提示「未找到该频道」或「文件不存在」。
  - 400（参数错误/格式不支持）：提示具体原因，如「仅支持 .txt、.md、.docx」。
  - 503（数据库不可达）：提示「服务暂时不可用，请稍后再试」。

### 5.2 连接与实时性

- WebSocket 断开时，前端可静默重连或提示「连接已断开，正在重连…」；恢复后不再重复提示。
- 发送消息后若接口失败，保留输入框内容并提示「发送失败，请重试」，避免用户误以为已发出。

## 6. 术语与一致性

全系统（前端、API 文案、文档）统一使用下表用语：

| 概念 | 用语 | 避免使用 |
|------|------|----------|
| 多人在线协作的会话容器 | 频道 | 群组、房间、Channel（面向用户时） |
| 容纳多个频道的顶层单位 | 空间（Workspace） | Team、工作区 |
| 被 @ 触发的 AI 账号 | Bot | Agent、机器人（技术文档可保留 Agent） |
| 用户主动输入并发送的一段内容 | 消息 | 帖子、留言 |
| 四层记忆（ANCHOR/DECISIONS/FILES_INDEX/RECENT） | 频道上下文 / 项目记忆 | 仅技术文档用「Context Store」 |

## 7. 可访问性与兼容

- **浏览器**：支持主流现代浏览器（Chrome、Firefox、Safari、Edge 近期版本）；避免依赖非标准 API。
- **移动端**：布局可考虑响应式（M2 可加强），核心操作在窄屏下可用。
- **网络**：弱网下请求超时与 WebSocket 断线有明确反馈，不无限转圈。

## 8. 扩展点（后续里程碑）

| 项目 | 说明 |
|------|------|
| 管理后台向导 | M2 提供「创建空间」「创建频道」「添加 Bot 到频道」的步骤式向导，减少 API 直接操作 |
| @ 提及补全 | 输入 `@` 后列出可 @ 的 Bot 与成员，减少输入错误 |
| 首次访问引导 | 检测无数据时引导创建默认空间与示例频道，并可选发送示例消息 + @Bot |
| 帮助入口 | 界面固定位置提供「帮助」或「？」链接至《使用说明书》关键章节 |
| 键盘快捷键 | 可选：Ctrl+K 聚焦搜索/频道切换等，与主流聊天产品一致 |

## 9. 多人协作易用性设计

> 聚焦多用户同时使用时的协作体验。

### 9.1 权限管理体系

#### 9.1.1 角色定义

| 角色 | 权限范围 | 适用用户 |
|------|----------|----------|
| 系统管理员 | 全系统最高权限：用户管理、空间管理、频道管理、Bot配置、系统设置 | 系统运维人员 |
| 空间管理员 | 本空间内：成员管理、频道创建、Bot配置 | 团队负责人 |
| 频道管理员 | 本频道内：成员管理、频道信息编辑、置顶消息 | 项目负责人 |
| 成员 | 发送消息、上传文件、@Bot | 普通用户 |
| 访客 | 仅查看公开频道（不可发言） | 外部协作人员 |

#### 9.1.2 权限继承规则

- 系统管理员拥有全系统所有空间和频道的管理权限
- 空间管理员默认拥有其空间内所有频道的管理权限
- 新创建频道继承所属空间的成员列表

### 9.2 协作感知设计

#### 9.2.1 在线状态

| 状态 | 判定条件 | 界面展示 |
|------|----------|----------|
| 在线 | WebSocket 连接正常 | 绿色圆点 |
| 离开 | 无操作超过 5 分钟 | 黄色圆点 |
| 离线 | WebSocket 断开 | 灰色圆点 |
| 勿扰 | 用户主动设置 | 红色圆点 |

#### 9.2.2 已读未读

- 显示「XX 人已读」而非具体人员列表（保护隐私）
- 已读状态仅对自己可见，不公开
- 支持一键「全部标为已读」

### 9.3 消息协作功能

#### 9.3.1 引用回复

- 右键/长按消息 → 「引用回复」
- 被引用消息以折叠形式展示在输入框上方
- 支持多层嵌套引用（最多 3 层）

#### 9.3.2 @提及增强

| 提及类型 | 触发方式 | 通知行为 |
|----------|----------|----------|
| @单个用户 | @用户名 | 仅通知被@者 |
| @多个用户 | @用户1 @用户2 | 通知所有被@者 |
| @所有人 | @all / @channel | 通知频道全员（需权限） |
| @Bot | @Bot名 | 触发 Bot 响应 |

#### 9.3.3 频道公告

- 频道管理员可发布公告
- 公告固定显示在频道顶部（最新一条覆盖）
- 公告支持 Markdown 格式

### 9.4 成员管理界面

```
频道成员管理：
┌─────────────────────────────────────────────────┐
│  成员 (12)                    [+ 添加成员]       │
├─────────────────────────────────────────────────┤
│  👤 张三（你）                    频道管理员    │
│  👤 李四                          成员       ✏️ │
│  🤖 @codebot                      Bot         ✏️ │
│  👤 王五                          成员       ✏️ 🗑️│
│  ...                                         │
└─────────────────────────────────────────────────┘
```

### 9.5 错误反馈（协作场景）

| 场景 | 错误提示 | 操作建议 |
|------|----------|----------|
| 无权限操作 | 「只有频道管理员可以此操作」 | 提供「申请权限」入口 |
| 邀请链接失效 | 「该邀请链接已过期」 | 提供「重新生成」按钮 |
| 被移出频道 | 「你已被移出该频道」 | 显示原因，提供返回入口 |
| 成员已达上限 | 「该频道成员已达上限（100人）」 | - |

## 10. MVP 功能差异分析

> 为开发优先级提供决策依据。

### 10.1 MVP 定义

**MVP（最小可用产品）目标**：一个可交付给真实用户试用的版本，具备核心价值但功能不求完善。

**MVP 验收标准**：
1. 用户可注册登录并创建/加入频道
2. 频道内可发送消息并实时接收
3. 可添加至少一个 Bot 并通过 @ 触发响应
4. 管理员可通过 Web 界面完成基本配置
5. 系统可通过 Docker Compose 一键部署

### 10.2 功能清单与当前状态

| 功能模块 | 功能点 | 当前状态 | MVP 必要性 | 备注 |
|----------|--------|----------|------------|------|
| **用户系统** | 用户注册/登录 | ✅ 已实现 | 必须 | 基础中的基础 |
| | 权限控制（RBAC） | ⚠️ 基础实现 | 必须 | 需完善到可用 |
| | 邀请机制 | ❌ 缺失 | 可选（M2） | MVP 可手动添加 |
| **频道系统** | 创建/删除频道 | ✅ 已实现 | 必须 | - |
| | 频道成员管理 | ⚠️ 基础实现 | 必须 | 需完善增删改查 |
| | 公开/私密频道 | ⚠️ 缺失 | 可选 | MVP 默认为公开 |
| **消息系统** | 发送/接收消息 | ✅ 已实现 | 必须 | - |
| | WebSocket 实时 | ✅ 已实现 | 必须 | - |
| | 消息历史加载 | ⚠️ 基础实现 | 必须 | 需支持分页 |
| | 引用/回复 | ❌ 缺失 | 可选 | 讨论效率相关 |
| **Bot 系统** | Bot 注册 | ✅ 已实现 | 必须 | - |
| | @触发 Bot | ✅ 已实现 | 必须 | - |
| | Bot 响应展示 | ⚠️ 基础实现 | 必须 | 需优化展示效果 |
| | 多 Bot 协作 | ⚠️ 设计已有 | 可选（M2） | MVP 单 Bot 即可 |
| **记忆系统** | ANCHOR.md | ⚠️ 需完善 | 必须 | 需 UI 编辑入口 |
| | DECISIONS.md | ⚠️ 需完善 | 必须 | 需 UI 查看入口 |
| | FILES_INDEX | ⚠️ 需完善 | 可选 | MVP 可暂不实现 |
| | RECENT.md | ⚠️ 需完善 | 可选 | MVP 可暂不实现 |
| **文件系统** | 文件上传 | ✅ 已实现 | 必须 | - |
| | 格式转换 | ⚠️ 部分实现 | 必须 | 需扩展支持范围 |
| | 文件预览 | ❌ 缺失 | 可选 | MVP 可下载查看 |
| **管理后台** | Bot 添加向导 | ❌ 缺失 | 必须 | MVP 核心差异化 |
| | 系统状态监控 | ❌ 缺失 | 可选 | MVP 可命令行 |
| | 频道上下文管理 | ❌ 缺失 | 可选 | MVP 可手动修改 |

### 10.3 MVP 优先级矩阵

```
                    高业务价值
                        │
        ┌───────────────┼───────────────┐
        │               │               │
        │   核心功能    │   增强功能    │
        │   (必须实现)  │   (M2 实现)   │
        │               │               │
低技术  ────────────────┼───────────────  高技术
难度                        │   复杂度
        │               │               │
        │   基础体验    │   锦上添花    │
        │   (尽快实现)  │   (按需实现)  │
        │               │               │
        └───────────────┼───────────────┘
                        │
                    低业务价值
```

#### 第一优先级（MVP 必须）

| 功能 | 技术难度 | 工作量估算 | 依赖 |
|------|----------|------------|------|
| 用户注册登录 + RBAC | 中 | 2-3 天 | 现有 User 模型 |
| 频道成员增删改查 | 低 | 1-2 天 | 现有 Channel 模型 |
| 消息历史分页加载 | 低 | 1 天 | 现有 Message 接口 |
| Bot 状态与「正在处理…」展示 | 低 | 1-2 天 | 现有 Bot 消息渲染 |
| Bot 响应带角色标签 | 低 | 1 天 | 现有 Bot 消息渲染 |
| 记忆文件 UI 编辑入口 | 中 | 2-3 天 | 现有 MemoryManager |
| 频道记忆（查看工作记录） | 低 | 1-2 天 | 现有 MemoryManager |
| 空间记忆（查看全部频道工作记录） | 中 | 2-3 天 | 现有 MemoryManager |
| Bot 添加向导 | 中 | 3-5 天 | 新建 Admin 页面 |
| 消息引用/回复（含文件引用） | 中 | 2-3 天 | Message 模型扩展 |
| 文件预览 | 中 | 3-5 天 | 文件转换管道完善 |

#### 第二优先级（M2 实现）

| 功能 | 技术难度 | 工作量估算 | 依赖 |
|------|----------|------------|------|
| 系统消息（邀请加入频道等） | 低 | 1-2 天 | 现有消息系统扩展 |
| 在线状态/已读未读 | 中 | 2-3 天 | WebSocket 状态管理 |
| 多 Bot 协作编排 | 高 | 5-7 天 | AgentOrchestrator 完善 |
| 系统状态监控 | 低 | 2-3 天 | 现有日志/服务状态 |

#### 第三优先级（按需）

| 功能 | 技术难度 | 工作量估算 |
|------|----------|------------|
| 正在输入提示 | 低 | 1-2 天 |
| 频道公告 | 低 | 1-2 天 |
| 向量检索（RAG） | 高 | 5-7 天 |
| 移动端适配 | 中 | 3-5 天 |

### 10.4 MVP 验收检查清单

```
□ 用户可注册账号并登录
□ 用户可创建新频道
□ 用户可手动添加其他用户到频道
□ 频道内消息可实时收发
□ 消息历史可翻页加载
□ 可注册并添加 Bot 到频道
□ @Bot 可触发响应并展示
□ 管理员可通过 Web 向导添加 Bot
□ ANCHOR.md 可通过 UI 查看和编辑
□ 系统可通过 Docker Compose 一键部署
□ 核心操作有中文错误提示
```

### 10.5 与完整版本的差异

| 维度 | MVP | 完整版本（规划） |
|------|-----|------------------|
| Bot 数量 | 1-2 个 | 5-15 个 |
| 记忆系统 | 手动维护 ANCHOR | 全部自动摘要 |
| 协作感知 | 无 | 在线/已读/输入提示 |
| 邀请方式 | 手动添加 | 链接/邮箱/批量 |
| 权限管理 | 基础 RBAC | 细粒度权限 |
| 消息功能 | 基础消息 | 引用/回复/公告 |
| 部署方式 | Docker Compose | Kubernetes（可选） |
| 移动端 | 基础响应式 | 原生 App |

### 10.6 风险与建议

| 风险 | 影响 | 建议 |
|------|------|------|
| RBAC 实现不完善 | 权限控制失效，数据泄露 | MVP 前完成基础权限测试 |
| Bot 响应不稳定 | 核心价值无法体现 | 选择最稳定的 LLM 提供商 |
| 记忆系统缺失 UI | 用户不会使用上下文 | 提供模板和示例 |
| 部署体验不佳 | 用户流失 | 优先打磨部署文档和脚本 |
