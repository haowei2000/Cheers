# AgentNexus 安装部署说明

> 面向**部署与运维人员**：环境要求、Docker 与本地两种安装方式、配置、数据库迁移、首次初始化。不涉及日常使用与管理操作。

---

## 一、环境要求

| 项目 | 要求 |
|------|------|
| 操作系统 | macOS、Linux；Windows 建议 WSL2 或等效环境 |
| Docker 方式 | Docker 20.10+、Docker Compose v2+ |
| 本地方式 | Python 3.13+、Node.js 18+、SQLite 3 |
| 可选 | Redis 7+（异步队列等，Docker 默认包含） |
| 浏览器 | Chrome、Firefox、Safari、Edge 近期版本（访问前端用） |

---

## 二、安装方式一：Docker Compose（推荐）

### 2.1 启动服务

在项目根目录执行：

```bash
docker compose up -d
```

- **前端**：http://localhost:80  
- **后端 API**：http://localhost:8000  
- 数据持久化使用卷 **agentnexus_data**，重启不丢数据。

### 2.2 数据库迁移（首次或升级后）

若镜像未自动执行迁移，需在 backend 容器内执行：

```bash
docker compose exec backend sh -c "cd /app && alembic upgrade head"
```

具体路径以实际镜像为准；若数据目录挂载在 `/app/data`，迁移会作用到该库。

### 2.3 首次使用前初始化（二选一）

**方式 A：使用种子数据（推荐，便于开箱测试）**

在 backend 容器内执行（或启动后端时设置环境变量 `SEED_DATA=1` 自动执行）：

```bash
docker compose exec backend python -m app.db.seed
```

种子数据会创建：默认工作空间、**测试项目**、**引导 Bot（@引导）**、测试 Bot（@小助）、测试用户；打开前端即可看到「测试项目」，进入后输入 **@引导 怎么用** 可获取使用引导。

**方式 B：手动初始化**

1. **创建工作空间**：见 [系统管理说明书 - 创建工作空间](系统管理说明书.md#二如何创建工作空间与项目)。  
2. **创建项目**：按系统管理说明书「创建项目」调用 API 创建至少一个项目。  
3. 打开前端，应能在左侧看到项目列表。

## 三、安装方式二：本地安装（开发或非容器部署）

### 3.1 后端

```bash
cd backend
python -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

复制并编辑环境配置（见 §四）：

```bash
cp ../.env.example ../.env
```

启动后端：

```bash
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

### 3.2 前端

```bash
cd frontend
npm install
npm run dev
```

浏览器访问前端开发地址（如 http://localhost:5173）。若前端需请求后端，请配置代理或后端地址为 http://localhost:8000。

### 3.3 数据库迁移（首次或升级后）

在**项目根或 backend 目录**下（已激活虚拟环境）：

```bash
cd backend
alembic upgrade head
```

### 3.4 首次使用前初始化（二选一）

**方式 A：使用种子数据（推荐）**

在项目根目录、已激活 backend 虚拟环境时执行：

```bash
cd backend
python -m app.db.seed
```

会创建默认工作空间、测试项目、引导 Bot（@引导）、测试 Bot（@小助）等；或启动后端时设置环境变量 **SEED_DATA=1**，首次启动会自动执行种子。

**方式 B：手动初始化**

与 Docker 方式相同：先创建工作空间，再创建项目（见 [系统管理说明书](系统管理说明书.md)）。

## 四、配置说明

### 4.1 环境变量

在项目根目录创建 **.env**（可复制 **.env.example** 后修改）。常用项：

| 变量 | 说明 | 示例 |
|------|------|------|
| DATABASE_URL | 主业务库 SQLite 路径 | `sqlite+aiosqlite:///data/main.db` |
| SQLITE_CONTEXT_PATH | Context Store 独立库路径 | `data/context_store/context.db` |
| REDIS_URL | Redis 地址（可选） | `redis://localhost:6379/0` |
| DATA_DIR | 数据根目录（上传、转换等） | `data` |
| DEBUG | 调试模式 | `false` |

Docker 部署时上述变量可在 **docker-compose.yml** 的 **environment** 中覆盖。

### 4.2 路径说明

- **DATABASE_URL**、**SQLITE_CONTEXT_PATH**：可为相对路径（相对进程工作目录）或绝对路径。  
- Docker 下数据目录通常挂载到容器内固定路径（如 `/app/data`），需与 **DATA_DIR**、**DATABASE_URL** 等一致。

## 五、部署后检查

1. **健康检查**：访问 `http://localhost:8000/health`，应返回 `{"status":"ok"}`。  
2. **API 文档**：访问 `http://localhost:8000/docs` 可查看 Swagger。  
3. **前端**：访问前端地址，若已按系统管理说明书完成工作空间与项目初始化，左侧应显示项目列表。
