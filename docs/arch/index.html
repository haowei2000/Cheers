当前拓扑

平台侧 Rust Backend，bot task 经进程内 InProcessBotLocator 直接派发（R1-A，Redis 退出启动路径）；本地 connector 通过 Agent Bridge control/data WS 接入；ACP agent 使用 OpenCode（opencode acp，持久 stdio）；MCP server 为可选注入，当前需 HTTP 适配。

客户端

Browser / Mobile REST + browser WS，订阅频道并接收消息与流式 delta；终态帧背压时收 4408 关闭信号转 REST 补齐（R3）。
New Frontend (frontend/) From-scratch 重做：React 18 + Vite 6 + TypeScript + Tailwind + Zustand + TanStack Query。3-panel 布局，WS 实时消息，Markdown + 代码高亮，JWT 认证。

Rust Backend

REST API workspaces / channels / messages / bots / files / friends / capability。Mention 解析 + bot trigger 派发。
进程内 Dispatch (R1-A) bot task 经 InProcessBotLocator 直接 try_send 到 control WS；session 命中即在线，无需 Redis 往返。create_placeholder 原子 INSERT 单点定胜负（R5）。
WS Gateway + Agent Bridge browser fan-out（InProcessFanout，终态帧背压关连接 R3）、control/data WS、stream registry（含 claim_finalize 守卫）、supersede。
Resource API 统一权限校验的 channel reads / membership-role-gated writes / activity / fs。session_id 已移除。

外部 Agent

Connector (Rust daemon) AccountRuntime → BridgeSession (control+data WS) → BridgeRuntime (task 编排) → AcpAdapter (ACP stdio)。
OpenCode ACP /tmp/opencode acp — 持久 ACP stdio agent。支持 OAuth 认证，无需 API key。session/new → session/prompt → 流式 session/update。
agentnexus-mcp-server 18 个 MCP tools (channel info, messages, files, fs.*)。stdio 模式通过 loopback HTTP 调用 connector 的 ResourceReq 转发。OpenCode 需要 HTTP/SSE MCP 格式 (待适配)。
Loopback Resource Server connector 本地 HTTP endpoint。接收 MCP tool call → 转成 data WS resource_req → Backend resource::dispatch() → 返回。

已验证的运行流

经 E2E 验证的核心链路；R1-A 后第 3 步由进程内 locator 直接派发（不再走 Redis）。流程 2→4 的 seq 连续性、finalize 幂等已由 R4-2 集成测试钉死。

1 消息写入 验证成员、解析 mention、事务内行锁分配 channel_seq（gap-free）、插入消息。先落库再广播（I1）。
2 Bot trigger 解析 resolve_bot_triggers() 匹配 mention_ids → 查找频道成员 bot。
3 进程内 dispatch InProcessBotLocator 查 session 在线 → 原子创建占位（rows_affected==1 定胜负）→ dispatch_task 直接 try_send 到 control WS。
4 Control WS 接收 Connector 的 control WS 收到 task 帧 → 转发给 BridgeRuntime。
5 Prompt 构建 build_prompt() 注入 system contract + channel_id + channel_name + trigger message。
6 ACP session/prompt AcpAdapter 调 OpenCode ACP session/new → session/prompt (JSON-RPC stdio)。
7 流式 delta OpenCode 通过 session/update (agent_message_chunk) 流式返回；Backend 盖 seq 戳（I7），流式帧可丢。
8 Data WS done → finalize BridgeRuntime 回传 delta/done；handle_done 经 claim_finalize 守卫、行锁分配 seq、is_partial→FALSE，广播终态帧（不丢，I6）。

已经明确的架构方向

这些决策在当前代码或文档中已经形成约束。

外部 agent 优先

平台不依赖内置 runtime；外置 ACP agent (OpenCode / Claude / Codex) 通过同一 ACP 协议接入。

频道只转发 @

没有中央 coordinator 参与意图判断，路由规则可测试、可审计。

资源写走 membership role

Resource API 读需频道成员，写需 owner/admin/member role。旧 Grant 机制已移除。

MCP 注入归 connector

daemon 只管理 connector 生命周期；connector 负责 MCP server 注入 (当前需 HTTP 适配)。

Agent Bridge 是平台协议

服务端不直接说 ACP JSON-RPC；ACP 只在 connector runtime adapter 层出现。

单实例 + 进程内 (R1-A)

fan-out / bot 路由 / 流注册表全进程内；Redis 退出 fan-out 启动路径，redis_* parked 备 HA（R1-B / M4）。

前端为原地重建

直接在 frontend/ 内 from-scratch 重做；React 18 + Vite 6 + TS + Tailwind。

ACP agent = OpenCode

opencode acp 支持持久 stdio ACP + OAuth 认证，无需 API key。MCP 需 HTTP 适配。
