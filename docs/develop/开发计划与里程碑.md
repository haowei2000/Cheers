## 1. 三阶段路线图（交付物驱动，时间弹性）

| 阶段 | 核心交付物 | 验收标准 |
|------|------------|----------|
| **里程碑 1：核心链路** | Python ChatCore（频道+WebSocket）、单 OpenClaw Bot 接入、基础文件转 MD（txt/md/docx）、@mention 路由、四层记忆结构建立（可先手动维护）、React 前端基础版、Docker Compose 一键部署 | 用户在频道 @bot 能正确回复；上传 docx 后 Bot 能读取内容；前端可登录、发消息、看历史 |
| **里程碑 2：多 Agent 协作** | 3–5 个专业化 Bot 同时运行、MemoryManager 自动摘要更新、文件类型扩展（pdf/xlsx/图片）、前端 Bot 标识与上下文面板、管理员后台 Bot 添加向导 | 同一频道多 Bot 可分别被 @ 且共享项目上下文；管理员通过向导完成新 Bot 配置，无需命令行 |
| **里程碑 3：智能调度** | Coordinator 主控 Bot、Bot 间结构化通信、自动任务分解与分发、响应质量监控看板、ChromaDB 大文件检索（可选） | 用户仅 @coordinator 即可由主控 Bot 分配任务并汇总结果 |

## 2. 门户阶段一实施顺序（建议）

门户阶段一任务可穿插在现有 M2/M3 之后实施，建议顺序：

1. Orchestrator 作为系统内置 Bot（与引导 Bot 并列）
2. Orchestrator 触发模式配置（必须 @ / 直接回答未 @，可关闭）
3. 引导 Bot 职责明确（仅系统使用问题，必须 @ 才活跃）
4. Bot 层级与 @ 规则（内置可 @ Bot/人，外部仅可 @ 人）
5. 自动接手机制与 UI（建议后显示「正在处理...」）
6. 所有 Bot 支持澄清
7. 资源监控基础版（队列、耗时、成功率）

## 3. TDD 策略

- **先写测试的范围**：ChatCore 频道/消息/成员 CRUD；MemoryManager 四层读写与拼接；BotAdapter 的 execute/health_check 契约；FileProcessor 各格式转 MD（可 mock 外部依赖）；@mention 解析与路由。
- **后补或集成测试**：WebSocket 连接与广播；端到端「发消息 → Bot 回复」；文件上传与状态机。
- **冻结规则**：进入 TDD 后不删减/弱化断言；怀疑测试有误时先说明原因，获用户确认后再改测试；业务代码迁就测试接口（方法名、入参出参）。
- **覆盖率目标**：业务核心模块 ≥ 80%。

## 5. 技术架构要点（实施时遵守）

- **数据存储**：主业务库与 Context Store 均使用 **SQLite**（主库单文件如 `data/main.db` 且启用 **WAL 模式**，Context Store 使用**独立**数据库文件如 `data/context_store/context.db` + MD 双轨）；当前无 PostgreSQL；当系统规模超出目标时，主业务库可迁移到 PostgreSQL，迁移路径见《关键技术文档》§5。
- **中文消息搜索**：规划采用 **jieba 分词预处理 + FTS5** 实现；M1 仅提供按频道列表消息，全文检索在 M2 或后续实现。
- **Context Store**（ADR D-02）：SQLite（WAL）+ MD 双轨；读 SQLite，写先 SQLite 再异步同步 MD。
- **多 Bot 执行**（ADR D-03）：默认串行；前端展示响应队列与预估时间。
- **Bot 审核**（ADR D-04）：仅拦截对外执行类高风险操作（规则匹配）。
- **OpenClaw 兼容**（ADR D-05）：Orchestrator 只依赖 BotAdapter 抽象接口；升级只改 Adapter。
- **系统 LLM**（附件一）：独立 API Key、异步调用、5 类场景不阻塞主消息流。

## 6. 后续里程碑设计依据（附件三、附件四）

- **附件三（人机协作架构）**：意图实时同步（/anchor、/decide、/redirect、/pause、/resume、/status）、知识确认机制、Task 生命周期与 Human-in-the-loop 确认节点、Critique Loop、自主错误恢复、Task DAG 与风险分级。M2 多 Agent 协作可引入意图同步与知识确认；M2/M3 可引入 Task 与 Human-in-the-loop、Critique Loop 与自主错误恢复。详见 `docs/develop/附件三_人机协作架构摘要.md`。
- **附件四（Agent 自主加入协议）**：A2A 握手、Agent Card、Receptionist Agent、七步握手、信任等级、`.well-known/agent-card.json`、`/a2a/enroll` WebSocket。M2 或 M3 可引入 Agent 自主加入协议与 Receptionist，作为 Bot 接入与信任体系扩展。详见 `docs/develop/附件四_Agent自主加入协议摘要.md`。
