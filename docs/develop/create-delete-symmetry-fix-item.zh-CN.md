# 创建/删除对称性修复事项

> **语言**：中文 | [English](create-delete-symmetry-fix-item.md)

## UX-DEL-001：补齐用户创建资源的删除路径

**类型**：用户体验 + 后端清理  
**优先级**：P1  
**状态**：已完成  

## 问题

系统中有些用户可见流程可以创建或上传资源，但没有对应的删除、移除或明确的保留说明。用户误操作后难以撤回，也会在文件库、消息历史、头像对象存储和 Agent Bridge 会话视图中留下过期数据。

## 本次修复结果

- 已补齐文件删除/解绑 API 与个人文件库、频道 Files、文件预览面板删除入口。
- 已补齐消息软删除/墓碑 API，并通过 WebSocket 同步客户端；普通消息、话题、公告均提供删除入口。
- 已补齐用户、Bot、工作空间头像删除 API；托管头像会同步清理对象存储，外链头像仅清空 URL。
- 已补齐 Agent Bridge Bot 会话关闭 API 与管理 UI。
- 已补齐账号自助停用流程，清理邮箱验证码、Keychain、好友关系与托管头像，并保留消息归属引用。

## 当前检查结果

已具备删除能力：

- 工作空间：`POST/DELETE /api/v1/workspaces/{workspace_id}`
- 频道与频道成员：`POST/DELETE /api/v1/channels`、`POST/DELETE /members`
- Bot、模型、模板：后端和设置页都有创建、更新、删除；内置对象不可删除属于明确保护。
- 记忆条目与待办：后端和 UI 均有创建、更新、删除。
- 密钥、公告 Issue、好友与黑名单：后端和 UI 均有创建/删除路径。

本次修复前确认的缺口：

| 范围 | 创建路径 | 当前删除缺口 |
|---|---|---|
| 上传文件 | `POST /api/v1/files/presign`、对象存储 `PUT`、`POST /api/v1/files/{file_id}/confirm`；Agent Bridge 文件上传也会创建 `FileRecord` | 没有 `DELETE /api/v1/files/{file_id}` 或解绑接口。个人文件库和频道 Files 面板只有预览/下载。发送前的待上传附件可移除，但已确认文件用户无法删除。 |
| 消息、话题、公告、转发消息 | `POST /api/v1/channels/{channel_id}/messages`、`/messages/forward`、公告发布组件 | 没有消息删除/墓碑接口。用户误发普通消息、话题、公告或转发后不能移除。 |
| 上传头像 | `POST /api/v1/avatars/users/me`、`/bots/{bot_id}`、`/workspaces/{workspace_id}` | UI 的 Clear 只把 `avatar_url` 保存为空，没有头像删除接口，也没有显式删除已上传的存储对象。 |
| Agent Bridge 会话 | 由 Agent Bridge scope 映射创建/轮换，并展示在会话面板 | 管理 UI 可以查看会话、刷新 DM 会话，但不能直接关闭或清理陈旧会话；实现前需要先定策略。 |
| 用户账号 | `POST /api/v1/auth/register` | 没有自助删除/停用账号流程；建议作为隐私与数据保留策略单独决策。 |

## 建议修复范围

1. **文件**
   - 增加后端文件删除/解绑 API。
   - 允许上传者、频道管理员或系统管理员从个人文件库或频道作用域移除文件。
   - 安全保留消息历史：如果文件仍被消息或多个作用域引用，则仅从当前作用域解绑或提示冲突；只有没有活跃引用时才硬删除物理对象。
   - 复用或抽取 `FileRetentionService` 中的物理文件/对象清理逻辑。
   - 在个人文件库、频道 Files 面板、文件预览面板增加删除入口。

2. **消息**
   - 增加 `DELETE /api/v1/channels/{channel_id}/messages/{msg_id}`。
   - 优先采用软删除/墓碑，而不是直接删行，避免破坏回复、话题、未读数、历史分页和 Bot trace。
   - 权限规则：发送者可删除自己的消息；频道管理员/系统管理员可删除任意非系统消息。Bot 消息是否允许 Bot owner 或频道管理员删除需明确。
   - 通过 WebSocket 广播消息更新/删除事件，并更新前端消息 store。
   - 给普通消息、话题根消息/回复、公告、转发包补齐删除入口。

3. **头像**
   - 增加 `DELETE /api/v1/avatars/users/me`、`/bots/{bot_id}`、`/workspaces/{workspace_id}`。
   - 使用与上传一致的权限检查。
   - 当当前头像 URL 指向系统托管头像路由时，同时清空 `avatar_url` 并删除固定头像对象。
   - 现有 Clear 按钮：托管上传头像调用删除接口；外链头像继续只清空 URL。

4. **策略落地项**
   - Agent Bridge 会话采用“关闭会话”：关闭后标记为 `closed`，并脱离活跃 scope 绑定，保留历史审计。
   - 账号采用“停用账号”：清理个人敏感数据并禁止登录，消息行保留墓碑式归属引用。

5. **文档**
   - 更新用户/管理员手册，说明删除行为、权限边界和保留例外。

## 验收标准

- 每个用户可见的创建/上传动作，要么有对应删除/移除动作，要么有明确文档说明为什么不可删除。
- 删除动作有确认提示，并提供成功/失败反馈。
- 后端权限测试覆盖所有者/上传者、频道管理员、系统管理员和无关用户。
- 文件删除不会导致消息渲染损坏，也不会遗留无引用的存储对象。
- 消息删除通过墓碑保留会话结构，并能实时同步到客户端。
- 头像删除同时清理数据库 URL 状态和系统托管的存储对象。
